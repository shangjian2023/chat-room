# Web 应用开发实验 - 项目源码

## 项目信息
- **实验名称**: 基于 Flask 的课程平台开发
- **技术栈**: Flask + MySQL + SQLAlchemy + Flask-Session
- **端口**: 8080

## 快速开始

### 前提条件
1. Python 3.8+ 已安装
2. MySQL 数据库已安装并运行

### 步骤 1: 安装依赖
```bash
cd course_platform
pip install -r requirements.txt
```

### 步骤 2: 配置数据库
修改 `setup_database.py` 中的 MySQL 密码：
```python
MYSQL_PASSWORD = '你的密码'  # 默认：123456
```

运行数据库初始化脚本：
```bash
python setup_database.py
```

### 步骤 3: 启动服务器
双击 `start.bat` 或运行：
```bash
python app.py
```

### 步骤 4: 访问应用
- 本地访问：http://localhost:8080
- 注册页面：http://localhost:8080/register
- 登录页面：http://localhost:8080/login

## 常见问题

### Q1: 数据库连接失败
**解决**:
- 确认 MySQL 服务已启动
- 检查用户名密码是否正确
- 确认端口 3306 未被占用

### Q2: 端口 8080 被占用
**解决**: 修改 app.py 第 143 行，更换端口：
```python
app.run(host='0.0.0.0', port=8081, debug=True)
```

### Q3: 无法跨终端访问
**解决**:
- 关闭防火墙或添加 8080 端口例外
- 使用本机 IP 地址访问（非 localhost）
- 查看 IP: `ipconfig`

## 实验原理总结

1. **HTTP POST 请求**: 表单数据通过请求体传输，适合敏感信息
2. **Flask 路由**: 通过 `@app.route` 装饰器绑定 URL 和处理函数
3. **SQLAlchemy ORM**: 面向对象操作数据库，无需编写 SQL
4. **Session 会话**: 服务器端存储用户状态，Cookie 传递 Session ID
5. **密码哈希**: 单向加密，不可逆，保障密码安全
